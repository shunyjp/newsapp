# NotebookLM Export Format v1

This document defines the canonical NotebookLM export format for this project.

- Status: fixed for production use
- Schema version: `notebooklm-pack.v1`
- Scope: JSON knowledge-pack export generated by `outputs/export_notebooklm.py`
- Rule: each YouTube video is exported as one `documents[]` entry

## Top-Level Structure

| Path | Type | Description |
|---|---|---|
| `schema_version` | `string` | Fixed schema identifier. Always `notebooklm-pack.v1`. |
| `generated_at` | `string` | Export generation timestamp in ISO-like string form. |
| `query` | `string \| null` | Query used for the run. May be `null` for channel-based runs. |
| `stats.video_count` | `int` | Number of exported videos. |
| `stats.available_count` | `int` | Number of videos with extracted content. |
| `stats.metadata_only_count` | `int` | Number of metadata-only unavailable videos. |
| `documents` | `list[object]` | One document per video. |

## Document Structure

Each `documents[]` item must contain exactly these sections:

- `video`
- `retrieval`
- `summary`
- `analysis`
- `evidence`

### `video`

| Path | Type | Description |
|---|---|---|
| `documents[].video.video_id` | `string` | YouTube video ID. |
| `documents[].video.title` | `string` | Video title. |
| `documents[].video.channel` | `string` | Channel name. |
| `documents[].video.published_at` | `string` | Published time as retrieved. |
| `documents[].video.url` | `string` | Watch URL. |

### `retrieval`

| Path | Type | Description |
|---|---|---|
| `documents[].retrieval.transcript_source` | `string` | Content source such as `manual`, `auto`, `description`, `api_description`, `none`, or `cached`. |
| `documents[].retrieval.transcript_length` | `int` | Extracted content length. |
| `documents[].retrieval.content_status` | `string` | Current values are `available` or `unavailable`. |
| `documents[].retrieval.content_warning` | `string` | Warning text for missing or degraded content. |
| `documents[].retrieval.is_metadata_only` | `bool` | `true` when content is unavailable and only metadata is retained. |

### `summary`

| Path | Type | Description |
|---|---|---|
| `documents[].summary.short_summary` | `string` | Short summary for quick scan. |
| `documents[].summary.detailed_summary` | `string` | Detailed summary text. |
| `documents[].summary.why_it_matters` | `string` | Why the item matters. |
| `documents[].summary.signal_score` | `float` | Aggregate importance score. |

### `analysis`

| Path | Type | Description |
|---|---|---|
| `documents[].analysis.reader_points` | `list[string]` | Reader-oriented key points. |
| `documents[].analysis.aggregated_key_points` | `list[string]` | Aggregated key points across chunks. |
| `documents[].analysis.aggregated_entities` | `list[string]` | Aggregated entities across chunks. |
| `documents[].analysis.aggregated_categories` | `list[string]` | Aggregated categories across chunks. |

### `evidence`

| Path | Type | Description |
|---|---|---|
| `documents[].evidence.chunk_count` | `int` | Number of evidence chunks. |
| `documents[].evidence.chunks` | `list[object]` | Chunk-level evidence array. |
| `documents[].evidence.cleaned_text` | `string` | Cleaned transcript or cleaned fallback description text. |

### `evidence.chunks[]`

| Path | Type | Description |
|---|---|---|
| `documents[].evidence.chunks[].chunk_id` | `string` | Chunk ID. |
| `documents[].evidence.chunks[].chunk_no` | `int` | Chunk order, starting at 1. |
| `documents[].evidence.chunks[].signal_score` | `float` | Chunk importance score. |
| `documents[].evidence.chunks[].summary` | `string` | Chunk summary text. |
| `documents[].evidence.chunks[].key_points` | `list[string]` | Chunk key points. |
| `documents[].evidence.chunks[].entities` | `list[string]` | Chunk entities. |
| `documents[].evidence.chunks[].categories` | `list[string]` | Chunk categories. |
| `documents[].evidence.chunks[].text` | `string` | Chunk text. |

## Normalization Rules

- Export must preserve the v1 shape even when source values are missing.
- String fields are normalized to strings.
- Integer fields are normalized to integers.
- Float fields are normalized to floats.
- List fields are normalized to `list[string]`.
- Missing-content videos must remain in `documents[]`; they must not be dropped.
- Missing-content videos must not fabricate chunks, entities, or key points.

## Compatibility Rule

- The canonical JSON layout is `documents[]`.
- Older top-level `videos[]` JSON is not the current format and should not be used for new integrations.
